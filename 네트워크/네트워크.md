# 네트워크

## 네트워크 개요

### 네트워크의 정의와 필요성

- 데이터를 송신자에서 수신자로 전달하기 위한 시스템. 여러 장치들이 연결되어 정보를 교환하는 방식으로, 현대의 컴퓨터와 통신 환경에서 필수적인 역할을 한다.
- **필요성:**
    - **정보 공유:** 데이터를 다양한 장치 간에 공유하여 효율적인 작업 환경을 제공한다.
    - **효율적 자원 활용:** 네트워크를 통해 프린터, 서버 등 자원을 공유하여 비용을 절감한다.
    - **의사소통 및 협업 지원:** 이메일, 메시지 등 의사소통 방식을 통해 협업을 촉진한다.
- **예시)**
    - **LAN:** 사무실 내에서 여러 컴퓨터가 연결되어 프린터를 공유하거나 파일 서버에 접근.
    - **WAN:** 전 세계적으로 이메일 송수신 및 웹사이트 접근을 통해 정보를 교환.

### 데이터 통신의 기본 구성 요소

- **송신자(Sender):** 데이터를 전송하는 장치
    - **예시)** 컴퓨터에서 웹 페이지를 요청하는 경우, 컴퓨터는 송신자가 된다.
- **수신자(Receiver):** 데이터를 받는 장치
    - **예시)** 웹 서버가 수신자가 되어 웹 페이지를 클라이언트에 전송한다.
- **메시지(Message):** 송수신되는 실제 데이터
    - **예시)** 이메일 본문, 첨부파일이 메시지이다.
- **전송 매체(Medium):** 데이터를 전달하는 경로
    - **예시)** Wi-Fi, 이더넷 케이블, 광섬유 등이 전송 매체이다.
- **프로토콜(Protocol):** 데이터 통신 규칙을 정의한 표준. 이를 통해 장치들이 서로 호환되며 통신을 할 수 있다.
    - TCP/IP, HTTP, FTP 등이 데이터 송수신을 규정하는 프로토콜이다.

### 네트워크 모델 (Network Models)

- **OSI 7계층 모델:**
    - **Physical (물리 계층):** 실제 전송 매체와 물리적인 연결을 다룬다. (예: 케이블, 신호)
    - **Data Link (데이터 링크 계층):** 데이터 프레임을 생성하고 오류를 감지하여 물리 계층에 전달한다. (예: MAC 주소)
    - **Network (네트워크 계층):** 데이터 패킷을 라우팅하고, IP 주소를 관리한다. (예: 라우터)
    - **Transport (전송 계층):** 데이터의 신뢰성을 보장한다. (예: TCP, UDP)
    - **Session (세션 계층):** 세션의 시작과 종료를 관리한다. (예: 로그인 유지)
    - **Presentation (표현 계층):** 데이터를 암호화하거나 압축하는 역할을 한다. (예: SSL 암호화)
    - **Application (응용 계층):** 사용자와 직접 상호작용하는 응용 프로그램을 제공한다. (예: HTTP, FTP)
    - 웹 요청 시, **HTTP** → **TCP** → **IP** → **Ethernet** 계층 순으로 데이터를 전달한다.
- **TCP/IP 모델:**
    - **Application:** 사용자와 상호작용하는 응용 계층 (예: HTTP, FTP)
    - **Transport:** 데이터의 신뢰성과 흐름을 관리하는 계층 (예: TCP, UDP)
    - **Internet:** 패킷을 라우팅하고 IP 주소를 관리하는 계층 (예: IP)
    - **Network Interface:** 실제 네트워크 인터페이스와 관련된 하드웨어 및 전송 계층 (예: Ethernet)

### 네트워크의 종류 (Types of Networks)

- **LAN (Local Area Network):**
    - **설명**: 작은 지역 내에서 컴퓨터와 장치들이 연결된 네트워크 (예: 집, 사무실).
    - **예시**: 회사 내 사무실 네트워크에서 직원들이 파일 공유 및 프린터를 사용.
- **WAN (Wide Area Network):**
    - **설명**: 넓은 지역, 예를 들어, 도시나 국가를 포함한 네트워크.
    - **예시**: 인터넷을 통한 이메일 송수신, 원격 사무실 연결.
- **MAN (Metropolitan Area Network):**
    - **설명**: 도시 범위의 네트워크로 LAN과 WAN의 중간 크기.
    - **예시**: 도시 내 공공기관들이 연결된 네트워크.
- **PAN (Personal Area Network):**
    - **설명**: 개인 장치들 간의 네트워크 (예: 스마트폰과 이어폰 연결).
    - **예시**: 블루투스를 통해 스마트폰과 무선 헤드폰 연결.
- **WLAN (Wireless LAN):**
    - **설명**: 무선 LAN, Wi-Fi 네트워크.
    - **예시**: 집이나 카페에서 Wi-Fi를 통해 인터넷에 접속.
- **SAN (Storage Area Network):**
    - **설명**: 대규모 데이터 저장 장치 간에 연결된 네트워크.
    - **예시**: 대형 데이터 센터에서 스토리지 장치들이 연결된 네트워크.

### 네트워크 토폴로지 (Network Topology)

장치들이 어떻게 물리적 혹은 논리적으로 연결될지를 정의하는 방식

- **스타(Star):** 중앙 노드가 모든 장치를 연결한다.
    - **장점:** 중앙 노드가 장애가 나지 않으면 네트워크의 다른 장치들은 영향을 받지 않는다.
    - **단점:** 중앙 노드가 고장 나면 전체 네트워크가 중단된다.
    - **예시)** 사무실 네트워크에서 스위치나 허브가 중앙 노드가 되어 컴퓨터들을 연결한다.
- **버스(Bus):** 모든 장치가 단일 케이블을 통해 연결된다.
    - **장점**: 설치가 간단하고 비용이 저렴하다.
    - **단점**: 충돌이 발생할 수 있고, 네트워크가 커질수록 성능 저하가 발생한다.
    - **예시)** 작은 네트워크에서 사용되는 단순한 연결 방식
- **링(Ring):** 모든 장치가 원형으로 연결되어 데이터가 한 방향으로 순차적으로 흐른다.
    - **장점**: 충돌이 적고 데이터 흐름이 예측 가능하다.
    - **단점**: 하나의 장치 또는 연결이 고장 나면 전체 네트워크가 영향을 받는다.
    - **예시)** 오래된 네트워크에서 흔히 사용되던 방식
- **메시(Mesh):** 모든 장치가 서로 연결되어 데이터를 여러 경로로 전달할 수 있다.
    - **장점**: 높은 안정성과 신뢰성을 보장한다.
    - **단점**: 많은 연결이 필요하여 설치와 유지 관리가 복잡하고 비용이 크다.
    - **예시)** 대형 데이터 센터나 고속 인터넷 서비스에서 사용
- **트리(Tree):** 계층 구조로 연결된 방식, 스타와 버스 토폴로지를 결합한 형태이다.
    - **장점**: 확장성이 좋고 중앙 관리가 용이하다.
    - **단점**: 트리의 가장 상위 노드가 고장 나면 전체 네트워크가 영향을 받을 수 있다.
    - **예시)** 기업의 네트워크에서 여러 부서가 각기 다른 스타 토폴로지로 연결되고, 상위 노드로 트리 형식의 네트워크가 구축된다.

## 데이터 통신 및 전송 기술

### 데이터 표현과 신호 (Data Representation and Signals)

- **디지털 데이터**: 0과 1의 신호를 이용하여 데이터를 표현한다.
    - **예시)** 컴퓨터의 모든 데이터는 이진 데이터로 표현된다.
- **아날로그 신호**: 주파수, 진폭, 위상 등의 아날로그 요소를 이용하여 신호를 표현한다.
    - **예시)** 전화 신호는 아날로그 방식으로 전송된다.

### 부호화와 변조 (Encoding and Modulation)

- **라인 부호화**: 디지털 데이터를 아날로그 신호로 변환하는 기술이다.
    - **예시)**
        - **NRZ (Non-Return-to-Zero)**: 0과 1을 각각 다른 전압으로 표현한다.
        - **Manchester 코드**: 각 비트가 중간에 전압 변화를 가진다.
- **변조:** 아날로그 신호를 변형하여 데이터를 전송하는 방식이다.
    - **FM (Frequency Modulation, 주파수 변조)**: 신호의 주파수를 변경하여 정보를 전달한다.
        - **예시**: FM 라디오 방송.
    - **AM (Amplitude Modulation, 진폭 변조)**: 신호의 진폭을 변경하여 정보를 전달한다.
        - **예시**: AM 라디오 방송.
    - **PM (Phase Modulation, 위상 변조):** 신호의 위상을 변경하여 정보를 전달한다.
        - **예시)** 일부 디지털 통신 시스템에서 사용된다.
- **다중화(Multiplexing):** 하나의 전송 매체에서 여러 신호를 동시에 전송하는 기술이다.
    - **TDM (Time Division Multiplexing, 시간 분할 다중화)**: 데이터를 시간 슬롯으로 나누어 여러 신호를 순차적으로 전송하는 방식. 각 사용자는 정해진 시간 슬롯을 독점적으로 사용하며, 동기화가 중요하다.
        - **특징**: 하나의 물리적 채널을 여러 사용자가 시간적으로 나누어 공유
        - **예시**: 한 회선에서 여러 전화 통화를 전송할 때 사용된다.
    - **FDM (Frequency Division Multiplexing, 주파수 분할 다중화)**: 각 사용자에게 서로 다른 주파수 대역을 할당하여 동시에 여러 신호를 전송하는 방식
        - **특징**: 각 주파수 대역이 겹치지 않도록 설계해야 하며, 대역폭이 제한적이다.
        - **예시**: 여러 TV 채널을 하나의 케이블을 통해 전송하는 방식
    - **CDM (Code Division Multiplexing, 코드 분할 다중화)**: 데이터를 서로 다른 코드로 구분하여 동일한 대역폭에서 여러 신호를 동시에 전송하는 방식. 모든 사용자가 동일한 주파수와 시간 자원을 공유하되, 고유한 코드를 사용하여 신호를 구분한다.
        - **특징**: 스펙트럼 효율이 높고, 여러 사용자가 동시에 데이터를 전송할 수 있음. 간섭이 적으며, 보안성이 상대적으로 높다.
        - **예시**: CDMA(코드 분할 다중 접속) 방식의 이동통신 기술. 여러 사용자가 동시에 동일한 주파수 대역을 사용하지만 고유 코드로 신호를 구분하여 통신.

### 전송 방식 (Transmission Methods)

- **직렬 전송**: 데이터가 한 번에 1비트씩 순차적으로 전송된다.
    - **예시**: USB는 직렬 전송 방식을 사용하여 데이터를 전송한다.
- **병렬 전송**: 여러 비트를 동시에 전송한다.
    - **예시**: 컴퓨터와 프린터 간의 병렬 포트는 병렬 전송을 사용하여 데이터를 전송한다.
- **동기식/비동기식 전송**:
    - **동기식 전송**: 전송 클럭에 동기화된 데이터를 전송한다.
        - **예시**: 네트워크에서 패킷이 일정 시간 간격으로 동기화되어 전송된다.
    - **비동기식 전송**: 각 데이터에 시작 비트와 정지 비트를 추가하여 전송한다.
        - **예시**: 키보드 입력에서 각 키 입력마다 비트가 비동기적으로 전송된다.

## 네트워크 프로토콜 (Network Protocols)

### 프로토콜의 개념과 역할 (Concept and Role of Protocols)

- **개념**: 네트워크 통신에서 데이터 전송과 수신을 위한 규칙과 절차를 정의한 표준이다.
- **역할**: 데이터를 송수신하는 방식과 순서를 규정하여 네트워크 장치 간의 원활한 통신을 보장한다.

### 데이터 링크 계층 프로토콜 (Data Link Layer Protocols)

- **이더넷(Ethernet)**: 가장 일반적인 LAN(로컬 영역 네트워크) 프로토콜로, MAC 주소를 사용하여 데이터를 전송한다.
    - **예시)** 집이나 사무실에서 사용하는 유선 네트워크.
- **PPP(Point-to-Point Protocol)**: 두 장치 간의 직렬 연결을 위한 프로토콜로, 인터넷 접속에 사용된다.
    - **예시)** 모뎀을 통해 인터넷에 연결할 때 사용된다.
- **HDLC(High-Level Data Link Control)**: 데이터 링크 계층에서의 오류 처리 및 흐름 제어를 위한 프로토콜로, 직렬 통신에 사용된다.
    - **예시)** 네트워크의 연결 상태를 감시하고, 오류를 처리하는 데 사용된다.

### 네트워크 계층 프로토콜 (Network Layer Protocols)

- **IP(Internet Protocol)**
    - 네트워크 상에서 장치들이 서로 데이터를 송수신할 수 있도록 주소를 지정하고, 패킷을 전송하는 역할을 한다.
    - 데이터를 패킷으로 나누어 전송한다.
    - 각 패킷에는 송신지와 수신지의 IP 주소가 포함된다.
    - 네트워크 간 경로를 계산하여 **패킷을 전달**한다.
        - 각 IP 패킷은 헤더와 데이터로 구성된다.
        - 헤더에는 송수신 IP 주소, 패킷의 크기, 프로토콜 정보 등이 포함된다.
        - 데이터는 실제 전송될 정보이다.
    - **IPv4**
        - **구조**: 32비트 주소 체계로, 4개의 옥텟(각각 8비트)으로 이루어져 있다. 각 옥텟은 0부터 255까지의 값을 가질 수 있다.
        - **헤더**: IPv4 헤더는 최소 20바이트로, 다음과 같은 필드를 포함한다.
            - **버전(Version)**: 4 (IPv4를 나타낸다)
            - **헤더 길이(Header Length)**: 헤더의 크기를 나타낸다.
            - **서비스 타입(Type of Service)**: 데이터의 우선순위 및 품질을 나타낸다.
            - **총 길이(Total Length)**: IP 패킷의 전체 크기 (헤더 + 데이터).
            - **식별자(Identification)**: 패킷을 구분하기 위한 값.
            - **플래그(Flags)**: 패킷 분할 여부를 표시한다.
            - **조각 오프셋(Fragment Offset)**: 패킷이 분할되었을 때 그 위치를 지정한다.
            - **TTL(Time to Live)**: 패킷이 네트워크를 통과할 수 있는 최대 홉 수.
            - **프로토콜(Protocol)**: 상위 계층 프로토콜을 지정한다 (예: TCP, UDP).
            - **헤더 체크섬(Header Checksum)**: 헤더 오류 검사.
            - **출발지 IP(Source IP)**: 패킷의 출발지 주소.
            - **목적지 IP(Destination IP)**: 패킷의 목적지 주소.
        - **주소 체계**: IPv4 주소는 32비트로 구성되며, 4개의 옥텟으로 구분된다.
            - **예시)** 192.168.0.1
    - **IPv6**:
        - **구조**: 128비트 주소 체계로, 8개의 16비트 섹션으로 구성된다. 각 섹션은 콜론(:)으로 구분된다.
        - **헤더**: IPv6 헤더는 40바이트로, IPv4보다 간단하고 효율적이고 다음과 같은 필드를 포함한다..
            - **버전(Version)**: 6 (IPv6를 나타낸다)
            - **트래픽 클래스(Traffic Class)**: 패킷의 우선순위를 나타낸다.
            - **흐름 레이블(Flow Label)**: 패킷이 같은 흐름에 속하는지 식별한다.
            - **페이로드 길이(Payload Length)**: 데이터 부분의 크기.
            - **다음 헤더(Next Header)**: 상위 계층 프로토콜을 식별한다.
            - **홉 제한(Hop Limit)**: 패킷이 네트워크를 통과할 수 있는 최대 홉 수.
            - **출발지 IP(Source IP)**: 패킷의 출발지 주소.
            - **목적지 IP(Destination IP)**: 패킷의 목적지 주소.
        - **주소 체계**: IPv6는 128비트 주소로, IPv4의 주소 부족 문제를 해결하기 위해 설계되었다. 주소는 8개의 16비트 그룹으로 나타내며, 불필요한 0은 생략 가능하다.
            - **예시)** 2001:0db8:85a3:0000:0000:8a2e:0370:7334
- **ICMP(Internet Control Message Protocol)**: 네트워크 상태를 점검하거나 오류 메시지를 전송하는 프로토콜이다.
    - **예시)** `ping` 명령어를 사용하여 네트워크 연결을 확인할 때 사용된다.
- **ARP(Address Resolution Protocol)**: IP 주소를 물리적 주소(MAC 주소)로 변환하는 프로토콜이다.
    - **예시)** 로컬 네트워크에서 다른 장치와 통신 시 목적지에 정확히 전달되기 위해 사용된다.
- **RARP(Reverse ARP)**: MAC 주소를 이용해 IP 주소를 찾는 프로토콜이다.
    - **예시)** 네트워크 부팅 시에 사용된다.

### 전송 계층 프로토콜 (Transport Layer Protocols)

- **TCP(Transmission Control Protocol)**: 신뢰성 있는 연결 지향 프로토콜로, 데이터의 송수신 상태를 추적하고 오류를 검출 및 수정한다.
    - **예시)** 웹 브라우징, 이메일 전송 등에서 사용된다.
- **UDP(User Datagram Protocol)**: 비연결성 프로토콜로, 빠른 전송을 위해 오류 검출 및 수정 기능을 제공하지 않는다.
    - **예시)** 실시간 영상 스트리밍, VoIP에서 사용된다.
- **공통점:**
    - **IP 프로토콜 기반**: TCP와 UDP는 모두 **IP(Internet Protocol)** 위에서 동작한다. 즉, 두 프로토콜은 데이터를 IP 패킷에 실어 전송하며, IP는 각 패킷을 목적지 주소로 라우팅한다.
    - **데이터 단위**: TCP와 UDP는 모두 **세그먼트(segment)**라는 데이터 단위를 사용하여 데이터를 전송한다. 세그먼트는 데이터와 함께 헤더를 포함하여 전송된다.
    - **포트 번호 사용**: TCP와 UDP는 모두 데이터를 전송할 때 **포트 번호**를 사용하여 송수신 애플리케이션을 구분한다. 포트 번호는 특정 서비스나 애플리케이션에 데이터가 전달되도록 돕는다.
    - **멀티플렉싱**: 두 프로토콜은 모두 여러 애플리케이션이 동일한 네트워크 연결을 공유할 수 있도록 멀티플렉싱(multiplexing)을 지원한다. 하나의 IP 주소에 여러 포트 번호를 매핑하여 다수의 애플리케이션이 동시에 데이터를 송수신할 수 있도록 한다.

### 응용 계층 프로토콜 (Application Layer Protocols)

- **HTTP/HTTPS**: 웹 브라우징을 위한 프로토콜로, HTTPS는 암호화된 연결을 제공한다.
    - **예시)** 웹 사이트 접속 시 사용된다.
- **FTP(File Transfer Protocol)**: 파일 전송을 위한 프로토콜로, 서버와 클라이언트 간에 파일을 주고받을 수 있다.
    - **예시)** 파일 서버에서 파일을 다운로드하거나 업로드할 때 사용된다.
- **SMTP/POP3/IMAP**: 이메일을 전송하고 받기 위한 프로토콜들이다.
    - **SMTP**: 이메일을 보내는 데 사용된다.
    - **POP3/IMAP**: 이메일을 수신하는 데 사용된다.
- **DNS(Domain Name System)**: 도메인 이름을 IP 주소로 변환하는 프로토콜이다.
    - **예시)** `www.example.com`을 `192.168.0.1`로 변환한다.
- **DHCP(Dynamic Host Configuration Protocol)**: 네트워크에서 동적으로 IP 주소와 기타 네트워크 설정 정보를 자동으로 할당하는 프로토콜이다.
    - **예시)** 라우터가 연결된 장치에 IP 주소를 자동으로 할당한다.

## 네트워크 장비 (Networking Devices)

### 스위치(Switch)와 허브(Hub)

- **스위치**: 네트워크에서 장치 간에 데이터를 효율적으로 전달하며, MAC 주소를 기반으로 데이터 전송 경로를 결정한다. 즉, 스위치는 데이터 패킷을 수신하고, 이를 목적지 MAC 주소에 맞는 포트로만 전달하여 불필요한 트래픽을 줄인다. 이로 인해 네트워크 성능이 향상된다.
    - **예시)** 사무실 내 유선 네트워크에서 여러 장치를 연결하는 데 사용된다.
- **허브**: 데이터를 수신하여 모든 장치에 전송하는 장치로, 데이터를 수신한 후 네트워크에 연결된 모든 장치로 데이터를 무차별적으로 전달한다. 이로 인해 네트워크 부하가 증가하고, 충돌이 발생할 가능성이 높다. 허브는 기본적인 네트워크 연결 기능만 제공하며, 데이터의 효율적 분배가 어렵다.
    - **예시)** 오래된 네트워크에서 사용되며, 오늘날에는 대부분 스위치로 대체된다.

### 라우터(Router)와 게이트웨이(Gateway)

- **라우터**: 서로 다른 네트워크를 연결하고, 데이터 패킷을 적절한 경로로 전달한다. 주로 IP 주소를 기반으로 경로를 결정하며, 서로 다른 서브넷 간의 트래픽을 라우팅한다. 라우터는 네트워크 간의 경로를 결정하고, 데이터를 효율적으로 전달하기 위해 라우팅 테이블을 사용한다.
    - **예시)** 집이나 기업 네트워크에서 인터넷에 연결할 때 사용된다.
- **게이트웨이**: 다른 네트워크 간의 프로토콜 변환을 담당하는 장치로, 다양한 네트워크를 연결한다.
    - **예시)** 사내 네트워크와 외부 인터넷 간의 연결을 위한 장치로 사용된다.

### 방화벽(Firewall)과 IDS/IPS

- **방화벽**: 네트워크 트래픽을 모니터링하고, 보안 정책에 맞지 않는 트래픽을 차단하는 장치이다. 방화벽은 들어오는 외부 트래픽과 내부 네트워크 사이의 보호 장치 역할을 한다. 방화벽은 IP 주소, 포트 번호, 프로토콜 등을 기준으로 필터링을 수행한다.
    - **예시)** 기업의 네트워크에 외부 공격을 방지하기 위해 사용된다.
- **IDS/IPS (Intrusion Detection System/Intrusion Prevention System)**: 네트워크에서 비정상적인 활동을 감지하고 경고를 보내는 시스템이며, IPS는 이를 한 걸음 더 나아가 비정상적인 활동을 실시간으로 차단하는 시스템이다.
    - **예시)** 해커의 침입을 감지하고 차단하는 시스템으로 사용된다.

### 무선 액세스 포인트(Wireless Access Point)

- **무선 액세스 포인트**: 무선 장치들이 네트워크에 연결할 수 있도록 신호를 송수신하는 장치이다. 액세스 포인트는 Wi-Fi 네트워크를 제공하며, 사용자들이 모바일 장치나 노트북 등을 무선으로 네트워크에 연결할 수 있게 해준다.
    - **예시)** Wi-Fi 네트워크를 제공하는 데 사용된다.
 
## 네트워크 설계와 관리 (Network Design and Management)

### 네트워크 설계 원칙 (Principles of Network Design)

- **확장성(Scalability)**: 네트워크가 성장할 때 새로운 장치나 트래픽 증가에도 성능이 유지되도록 설계한다.
    - **예시)** 대규모 회사의 네트워크 설계 시, 새로운 부서가 추가되어도 VLAN을 통해 간단히 확장 가능하도록 설정한다.
- **신뢰성(Reliability)**: 장애 발생 시 빠르게 복구할 수 있는 백업 장치와 이중화 구조를 포함해야 한다.
    - **예시)** 기업 데이터센터에서 이중 라우터와 이중 링크를 구성하여 장애 시에도 네트워크가 지속적으로 운영되도록 보장한다.
- **비용 효율성(Cost Efficiency)**: 높은 성능을 유지하면서도 비용을 줄이기 위한 기술과 장비를 선택한다.
    - **예시)** 소규모 사업체에서 관리형 스위치 대신 스마트 스위치를 선택하여 관리 기능과 비용의 균형을 맞춘다.
 
## 네트워크 설계와 관리 (Network Design and Management)

### 네트워크 설계 원칙 (Principles of Network Design)

- **확장성(Scalability)**: 네트워크가 성장할 때 새로운 장치나 트래픽 증가에도 성능이 유지되도록 설계한다.
    - **예시)** 대규모 회사의 네트워크 설계 시, 새로운 부서가 추가되어도 VLAN을 통해 간단히 확장 가능하도록 설정한다.
- **신뢰성(Reliability)**: 장애 발생 시 빠르게 복구할 수 있는 백업 장치와 이중화 구조를 포함해야 한다.
    - **예시)** 기업 데이터센터에서 이중 라우터와 이중 링크를 구성하여 장애 시에도 네트워크가 지속적으로 운영되도록 보장한다.
- **비용 효율성(Cost Efficiency)**: 높은 성능을 유지하면서도 비용을 줄이기 위한 기술과 장비를 선택한다.
    - **예시)** 소규모 사업체에서 관리형 스위치 대신 스마트 스위치를 선택하여 관리 기능과 비용의 균형을 맞춘다.

### IP 주소 체계 (IP Addressing System)

- **IPv4**
    - **구조**: 32비트 주소로 구성되어 있으며, 네트워크와 호스트를 구분한다.
        - **서브넷팅(Subnetting)**: 하나의 네트워크를 작은 네트워크로 분리하여 관리 효율성을 증가시킴.
            - **예시)** `192.168.1.0/24` 네트워크를 두 개의 서브넷 `/25`로 나누면 각각 128개의 IP를 제공하며, 두 부서로 나뉜 네트워크를 관리할 수 있다.
        - **CIDR(Classless Inter-Domain Routing)**: 클래스 개념 대신 주소를 효율적으로 나누는 방식.
            - **예시)** `192.168.1.0/22`는 4개의 `/24` 네트워크를 하나로 합쳐 관리한다.
- **IPv6**
    - **구조**: 128비트 주소 체계로, 8개의 16진수 블록으로 표현되며, 콜론으로 구분된다.
        - **특징**:
            - **방대한 주소 공간**: 거의 무한대에 가까운 주소를 제공.
            - **내장 보안**: IPsec이 기본적으로 포함되어 암호화와 인증 지원.
            - **자동 구성**: 장치가 네트워크에 연결될 때 자동으로 IP를 할당받음.
        - **예시)** `2001:0db8::/32` 주소 공간은 테스트 및 학습용으로 예약되어 사용 가능.

### VLAN과 네트워크 분리 (VLAN and Network Segmentation)

- **VLAN(Virtual Local Area Network)**: 하나의 물리적 네트워크 장비에서 여러 개의 논리적 네트워크를 생성하는 기술로, 네트워크 트래픽을 분리하고 보안을 강화한다. 물리적으로 동일한 스위치를 사용하더라도 VLAN을 설정하면 서로 다른 VLAN의 트래픽이 섞이지 않는다.
    - **특징**
        - 네트워크 분리: 동일한 네트워크 장치 내에서 논리적 네트워크 구분이 가능하다.
        - 보안 강화: 각 VLAN 간 트래픽이 격리되므로 중요 정보가 외부에 노출될 위험이 줄어든다.
        - 효율적인 관리: 부서별로 트래픽을 분리하여 대역폭을 효율적으로 사용할 수 있다.
    - **예시)**
        - IT 부서는 VLAN 10, HR 부서는 VLAN 20으로 설정하여 서로 다른 VLAN 간 트래픽이 섞이지 않도록 설정한다.
        - VLAN 10(IT 부서)은 주로 개발과 서버 유지보수를 위한 데이터 트래픽을 처리하며, VLAN 20(HR 부서)은 직원들의 일반적인 업무 데이터만 처리하도록 한다.
- **네트워크 분리의 필요성**:
    - VLAN을 활용하여 내부 네트워크와 외부 네트워크를 물리적으로 분리하거나, 게스트 사용자와 내부 직원의 트래픽을 분리함으로써 보안 사고와 성능 저하를 방지한다.

### NAT(Network Address Translation)와 포트 포워딩 (Port Forwarding)

- **NAT**: 사설 네트워크에 있는 장치들의 IP 주소를 공인 IP 주소로 변환하여 인터넷과 통신이 가능하도록 하는 기술이다. 이를 통해 **IP 주소 부족 문제를 해결**하고 내부 네트워크 장치를 **외부로부터 보호**한다.
    - **동작 원리**
        - 내부 네트워크에서 외부로 나가는 패킷의 소스 IP 주소를 공인 IP 주소로 변환.
        - 변환된 패킷에 매핑 정보를 저장.
        - 외부 네트워크에서 들어온 응답 패킷을 매핑 정보를 사용해 적절한 내부 장치로 전달
    - **SNAT(Source NAT)**: 내부 네트워크에서 외부로 나가는 트래픽의 소스 IP를 공인 IP로 변환한다.
        - **예시)** 내부 트래픽을 공인 IP로 변환해 인터넷 통신 가능하다.
    - **DNAT(Destination NAT)**: 외부에서 공인 IP로 들어오는 요청을 특정 내부 장치로 전달한다.
        - **예시)** 외부에서 웹 서버에 접속할 때 요청을 내부 IP로 전달한다.
- **포트 포워딩(Port forwarding)**: NAT의 기능 중 하나로, **외부 네트워크에서 특정 포트로 들어오는 요청을 내부 네트워크의 특정 장치로 연결**하는 방식이다.
    - **필요성**
        - NAT 환경에서는 외부에서 사설 IP 주소로 직접 접근이 불가능하다. 그러나, 웹 서버나 원격 데스크톱처럼 내부 장치에 접근이 필요한 경우 포트 포워딩을 통해 특정 요청을 전달한다.
    - **동작 원리**
        - 외부 사용자가 공인 IP와 특정 포트로 요청.
        - NAT는 요청을 미리 설정된 내부 네트워크 장치와 포트로 전달
        - 내부 장치에서 응답 시 NAT가 외부 사용자로 응답 반환
    - **예시)**
        - 웹 서버: 외부 사용자가 `203.0.113.10:80`으로 접속하면 요청을 내부 네트워크의 웹 서버(192.168.0.10:80)로 전달한다.
        - 원격 데스크톱: 외부에서 `203.0.113.10:3389`로 접속하면 요청을 내부 PC(192.168.0.20:3389)로 전달한다.

### 트래픽 관리와 QoS (Traffic Management and Quality of Service)

- **트래픽 관리**: 네트워크 장치에서 트래픽의 흐름을 모니터링하고 제어하여 병목 현상을 방지한다.
    - **트래픽 제어 방법**:
        - **트래픽 쉐이핑(Traffic Shaping)**: 네트워크 트래픽 속도를 제한하여 과도한 사용을 방지한다.
        - **로드 밸런싱(Load Balancing)**: 트래픽을 여러 서버로 분산하여 네트워크 부하를 줄인다.
        - **대역폭 할당(Bandwidth Allocation)**: 사용자 또는 애플리케이션별로 대역폭을 고정 또는 가변적으로 설정한다.
    - **예시)** 기업 네트워크에서 직원들이 스트리밍 서비스를 사용하는 것을 제한하고, 업무 관련 트래픽에 더 많은 대역폭을 할당.
- **QoS(Quality of Service)**: 네트워크 트래픽 중 중요 데이터를 우선적으로 처리하여 서비스 품질을 보장하는 기술이다.
    - **기능**
        - **우선 순위 부여(Prioritization)**: VoIP와 같은 실시간 트래픽을 중요하게 처리한다.
        - **트래픽 분류(Traffic Classification)**: 데이터 패킷을 유형별로 분류하여 대역폭을 효율적으로 사용한다.
        - **대기열 관리(Queuing)**: 패킷을 대기열에 적절히 분배하여 네트워크 병목 현상을 방지한다.
    - **예시)** 회사에서 VoIP 통화와 비디오 컨퍼런스 트래픽은 가장 높은 우선순위를 부여하며, 일반 인터넷 브라우징 트래픽은 낮은 우선순위를 부여한다.
 
## 라우팅 및 스위치 (Routing and Switching)

### 라우팅의 개념 (Routing Concepts)

네트워크에서 데이터를 최적 경로를 통해 목적지로 전달하는 과정

- **정적 라우팅(Static Routing)**: 관리자가 수동으로 경로를 설정하는 방식이다. 경로에 문제가 생길 경우 대처하기 어렵다.
    - **예시)** 소규모 네트워크에서 특정 목적지로만 데이터를 전송할 때 사용된다.
- **동적 라우팅(Dynamic Routing)**: 라우팅 프로토콜을 사용해 경로를 자동으로 설정하고 업데이트한다. 라우터가 판단하여 가장 효율적인 방법으로 패킷을 전송시킨다.
    - **예시)** 대규모 네트워크에서 장애 발생 시 자동으로 우회 경로를 설정한다.

### 라우팅 프로토콜 (Routing Protocols)

네트워크에서 데이터를 목적지까지 효율적으로 전달하기 위해 **경로를 설정하고 관리**하는 규칙

- **내부 라우팅 프로토콜 (Interior Gateway Protocol, IGP):** 같은 자율 시스템(AS, Autonomous System: 단일 관리 하에 있는 네트워크 그룹) 내에서 경로를 설정하고 관리하는 프로토콜
    - **RIP (Routing Information Protocol)**: 거리 벡터 기반 프로토콜로, 홉 수를 기준으로 경로를 설정한다.
        - **특징:** 단순하지만 홉 수가 15를 초과하면 도달할 수 없다.
        - **예시)** 소규모 네트워크에서 간단한 라우팅 설정 시 사용된다.
    - **OSPF (Open Shortest Path First)**: 링크 상태 기반 프로토콜로, **최단 경로 알고리즘**을 사용해 효율적 경로를 설정한다.
        - 특징: 대규모 네트워크에 적합하며, 계층적 설계가 가능하다.
        - **예시)** 기업 네트워크에서 경로 효율성을 높이기 위해 사용된다.
    - **EIGRP (Enhanced Interior Gateway Routing Protocol)**: 거리 벡터와 링크 상태의 장점을 결합한 프로토콜이다.
        - **특징**: 빠른 경로 재설정과 효율적으로 자원을 사용한다.
        - **예시)** 고성능 네트워크 환경에서 사용된다.
- **외부 라우팅 프로토콜 (Exterior Gateway Protocol, EGP):** 다른 자율 시스템(AS) 간의 경로를 설정하고 관리하는 프로토콜
    - **BGP (Border Gateway Protocol)**: 자율 시스템(AS) 간의 경로를 설정하며, 인터넷 라우팅에 사용된다.
        - **특징**: 경로 정책을 기반으로 경로를 선택하며, 높은 확장성과 유연성을 제공한다.
        - **예시)** 인터넷 서비스 제공업체(ISP) 간의 경로 설정에 사용된다.

### 스위치와 데이터 전달 (Switching and Data Transmission)

스위치는 네트워크 내부에서 **데이터 프레임을 전달**하는 장치로, **2계층 (데이터 링크 계층)**에서 동작한다.

- **MAC 주소 테이블 관리**: 스위치는 MAC 주소를 기반으로 데이터 프레임을 전송하며, MAC 주소 테이블을 유지한다.
    - **특징**: 네트워크 성능 향상과 충돌 최소화
    - **예시)** 사무실 네트워크에서 장치 간 데이터 전달 시 사용된다.
- **STP (Spanning Tree Protocol)**: 네트워크 루프를 방지하기 위해 논리적 트리를 구성한다.
    - **특징**: 중복된 경로가 있는 환경에서도 안정적인 데이터 전달 가능
    - **예시)** 다중 스위치가 연결된 네트워크에서 루프를 방지하기 위해 사용된다.
 
## 네트워크 보안 (Network Security)

### 네트워크 위협과 공격 (Network Threats and Attacks)

- **DoS/DDoS (Denial of Service/Distributed DoS)**: 네트워크나 시스템에 과도한 트래픽을 보내 서비스가 정상적으로 작동하지 못하게 만드는 공격
    - **DoS**: 하나의 소스에서 공격을 수행.
    - **DDoS**: 여러 분산된 장치(보통 봇넷)를 이용해 공격.
    - 공격 대상은 웹 서버, 이메일 서버, DNS 서버 등 다양한 시스템일 수 있고, 공격의 결과로 정상 사용자들이 서비스에 접속하지 못하게 된다.
    - **예시)** 공격자가 웹사이트를 다운시키기 위해 대량의 트래픽을 유발한다.
- **스니핑(Sniffing)**: 네트워크에서 데이터를 가로채는 공격이다.
    - 공용 Wi-Fi 네트워크에서 발생하기 쉽고 주로 패킷을 캡처해 암호화되지 않은 데이터(예: 로그인 정보, 이메일 내용 등)를 탈취한다.
    - **예시)** 카페의 공용 Wi-Fi에 연결된 사용자의 로그인 정보를 탈취
- **피싱(Phishing)**: 사용자를 속여 민감한 정보를 입력하도록 유도하는 사회 공학적 공격
    - 이메일, 문자, 가짜 웹사이트 등을 통해 사용자를 속이고, 주로 금융정보(계좌번호, 카드 정보)나 계정 로그인 정보를 노린다
    - **예시)** 은행을 사칭한 이메일로 비밀번호 재설정을 요구하며, 가짜 로그인 페이지로 유도한다.
- **MITM (Man-In-The-Middle)**: 통신 중간에서 데이터를 가로채거나 수정하는 공격이다.
    - 주로 암호화되지 않은 HTTP 트래픽에서 발생한다.
    - **예시)** 공용 Wi-Fi에서 사용자와 웹사이트 간의 트래픽을 가로챈다.

### 보안 기술 (Security Technologies)

- **암호화(Encryption)**: 데이터를 보호하기 위해 암호화 알고리즘을 사용한다.
    - **대칭키 암호화**
        - 하나의 키로 데이터를 암호화하고 복호화.
        - **특징**: 빠르지만, 키 분배가 어려움.
        - **예시)**
            - **AES(Advanced Encryption Standard):** DES의 안전성에 대해 여러가지 공격 방법들이 발표되며 미국의 NIST 기관에서 고안한 암호 알고리즘
            - **DES(Data Encryption Standard):** 평문을 64비트로 나누어 56비트 키를 이용해 다시 64비트의 암호문을 만들어내는 알고리즘
    - **비대칭키 암호화**
        - 공개키와 개인키를 사용하여 암호화/복호화.
        - **특징**: 키 분배는 쉽지만 속도가 느림.
        - **예시)**
            - **RSA(Rivest Shamir Adleman):** 암호화 하는 키값과 복호화 하는 키값이 서로 다르며, 따라서 두 가지 키를 가지고 암/복호화를 수행
            - ECC(효율적이고 강력한 보안 제공).
- **SSL/TLS**: 웹 통신을 암호화하여 안전한 데이터 전송을 보장하는 프로토콜
    - HTTPS(SSL/TLS를 사용하는 HTTP)는 웹사이트와 사용자 간 데이터를 암호화하여 전송 중 탈취를 방지한다.
    - **예시)** 은행 사이트에서 로그인할 때 `https://`로 시작하는 주소 표시
- **IPSec**: 네트워크 계층에서 데이터 전송을 암호화하고 인증하여 보안성을 높이는 프로토콜
    - 데이터를 전송하는 동안 도청 및 변조를 방지하고, VPN에서 주로 사용된다.
    - **예시)** 원격 근무자가 회사 서버에 안전하게 접속할 때 VPN을 사용하는 경우

### 방화벽과 ACL(Access Control List)

- **방화벽**: 네트워크 트래픽을 모니터링하고, 보안 정책에 따라 허용하거나 차단하는 시스템
    - 소프트웨어 기반 또는 하드웨어 기반으로 구현 가능하고 네트워크 외부(인터넷)와 내부(사설 네트워크) 간의 트래픽을 제어한다.
    - **예시)** 비정상적인 외부 트래픽(예: 특정 포트에서 반복되는 접속 시도)을 차단한다.
- **ACL (Access Control List)**: 네트워크 트래픽의 허용/차단 규칙을 정의하는 설정
    - IP 주소, 포트 번호, 프로토콜 등을 기준으로 트래픽을 제어한다.
    - **예시)** 회사에서 특정 직원의 PC만 외부 서버에 접근할 수 있도록 허용한다.

### VPN (Virtual Private Network)

공용 네트워크(예: 인터넷)를 통해 암호화된 통로를 만들어 안전하게 데이터를 전송하는 기술

- 데이터를 암호화하여 해커가 트래픽을 감시하거나 탈취하지 못하게 한다.
- 원격 근무, 기업 내부망 접속, 지리적 제한 우회 등에 사용된다.
- **예시)** 재택근무자가 회사의 내부 네트워크에 접속하여 업무 데이터를 안전하게 처리한다.

### Zero Trust 네트워크 (Zero Trust Network)

네트워크 내부와 외부를 구분하지 않고, 모든 사용자와 장치의 신뢰를 최소화하여 보안을 강화하는 모델

- 인증과 권한 부여를 엄격히 수행하며, 사용자와 장치의 동작을 지속적으로 모니터링한다.
- 클라우드 환경에서 효과적이며, 데이터 보호와 규제 준수를 지원한다.
- **예시)** 직원이 클라우드 기반 업무 앱에 접근하려면, MFA(다중 인증)와 장치 검증을 통과해야 한다.

## 네트워크 성능 평가 및 최적화 (Network Performance Evaluation and Optimization)

### 네트워크 성능 지표 (Network Performance Metrics)

- **대역폭 (Bandwidth)**
    - 네트워크가 처리할 수 있는 데이터의 최대 용량을 의미하며, 네트워크의 전송 용량을 나타낸다. 대역폭이 클수록 한 번에 전송할 수 있는 데이터 양이 많아지며, 이로 인해 빠른 데이터 전송 속도를 경험할 수 있다.
    - **단위:** Mbps, Gbps 등.
    - 대역폭이 높다는 것은 동시에 더 많은 데이터를 전송할 수 있다는 뜻이다. 예를 들어, 1Gbps의 대역폭을 가진 네트워크는 초당 최대 1Gb의 데이터를 전송할 수 있다.
    - **예시)**
        - 인터넷 서비스 제공업체가 제공하는 서비스 중 하나가 1Gbps 대역폭을 제공한다고 할 때, 이 네트워크는 초당 최대 1기가비트의 데이터를 전송할 수 있다.
        - 영상 스트리밍 서비스에서 고화질(4K) 영상을 버퍼링 없이 스트리밍하려면 고속 대역폭(최소 25Mbps 이상)이 필요하다.
- **지연 (Latency)**
    - 데이터가 출발지에서 목적지까지 이동하는 데 걸리는 시간이다. 이는 네트워크에서의 응답 속도와 직결되며, 특히 실시간 서비스에 중요한 영향을 미친다.
    - **단위:** 밀리초(ms).
    - 지연 시간이 낮을수록 데이터가 빠르게 전달되어 응답이 빠르게 이루어지며, 높은 지연은 네트워크 성능 저하를 의미한다.
    - **예시)**
        - 온라인 게임에서 50ms 이하의 지연 시간은 게임이 원활하게 진행되도록 돕지만, 200ms 이상의 지연이 발생하면 플레이어가 느리게 반응하고 랙이 발생한다.
        - 음성 통화나 영상 회의에서 지연 시간이 길어지면 대화의 흐름에 지장을 주며, 간헐적인 끊김이 발생할 수 있다.
- **처리량 (Throughput)**
    - 실제로 네트워크에서 전송된 데이터의 양을 나타낸다. 대역폭은 네트워크가 최대한 처리할 수 있는 용량을 의미하는 반면, 처리량은 실제 사용 중 전송된 데이터의 양을 나타낸다.
    - **예시)** 네트워크의 대역폭이 1Gbps라도 실제 처리량이 600Mbps일 경우, 네트워크 사용률은 60%로 계산된다. 이는 네트워크가 최대 용량의 60%만 활용되고 있음을 의미하며, 병목 현상이나 최적화가 필요할 수 있음을 나타낸다.

### 네트워크 트래픽 분석 (Traffic Analysis)

- **개념**: 네트워크에서 발생하는 데이터를 모니터링하고 분석하여 성능 문제를 해결하고, 최적화 방안을 찾는 과정이다. 이를 통해 비정상적인 트래픽을 감지하고, 문제를 미리 예방할 수 있다.
- **도구**
    - **Wireshark**: 네트워크 트래픽을 실시간으로 캡처하고 분석할 수 있는 도구.
    - **SolarWinds**: 네트워크 성능 모니터링 및 트래픽 분석 도구.
    - **NetFlow Analyzer**: 네트워크 트래픽 패턴을 분석하여 성능 최적화 및 용량 계획에 활용되는 도구.
- **예시)**
    - **트래픽 급증 분석**: 특정 시간대에 네트워크 트래픽이 급증하는 패턴을 발견했을 때, 해당 시간대에 어떤 애플리케이션이나 서비스가 원인인지를 분석하여, 성능 최적화를 위한 추가 용량을 배분하거나 서버 확장을 검토할 수 있다.
    - **DDoS 공격 대응**: 비정상적인 트래픽 패턴이 감지되면, 해당 패턴을 분석하여 공격의 원천을 차단하고, 공격을 완화하기 위한 대응을 진행할 수 있다.

### 부하 분산과 로드 밸런싱 (Load Balancing)

- **개념**: 여러 서버나 네트워크 장치에 트래픽을 분산시켜 과부하를 방지하고 시스템의 안정성을 높이는 기술이다. 이를 통해 트래픽이 특정 서버에 집중되는 것을 방지하고, 서버의 자원을 효율적으로 활용할 수 있다.
- **방법**
    - **라운드 로빈 (Round Robin)**: 트래픽을 순차적으로 서버에 분배하는 방식이다.
    - **최소 연결 (Least Connections)**: 가장 적은 연결 수를 가진 서버에 트래픽을 할당하는 방식이다.
    - **IP 해싱 (IP Hashing)**: 클라이언트의 IP 주소를 기준으로 트래픽을 분배하는 방식이다.
- **예시)**
    - **웹 서비스 로드 밸런싱**: 대규모 웹 서비스에서 사용자 수가 증가하면, 로드 밸런서를 통해 트래픽을 여러 서버로 분산하여 서버 과부하를 방지하고, 서비스 속도를 일정하게 유지할 수 있다.
    - **클라우드 환경에서의 최적화**: 클라우드 기반의 웹 서비스가 많은 사용자를 처리할 때, 다양한 인스턴스에 워크로드를 분배하여 성능을 최적화할 수 있다. 예를 들어, AWS Elastic Load Balancer(ELB)는 트래픽을 여러 EC2 인스턴스로 분산시켜 안정성을 보장한다.
