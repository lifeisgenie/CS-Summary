## 데이터베이스 개요

### 데이터베이스의 정의와 필요성

- **정의**: 데이터를 체계적으로 저장하고 관리하여 다수의 사용자가 데이터를 효율적으로 조회, 수정할 수 있도록 지원하는 시스템이다.
- **필요성**:
    - 데이터 중복 제거: 동일 데이터의 중복을 방지하여 저장소를 효율적으로 사용한다.
    - 데이터 일관성 유지: 모든 시스템에서 동일한 데이터 상태를 유지하도록 한다.
    - 데이터 무결성 보장: 데이터의 정확성과 신뢰성을 확보한다.
    - 데이터 공유 및 보안 강화: 다수 사용자의 접근을 제어하고 데이터를 보호한다.

### 파일 시스템과 데이터베이스의 차이

- 파일 시스템은 데이터의 비구조적 관리에 한정되며 중복, 무결성 문제를 초래한다.
- 데이터베이스는 데이터 모델을 통해 데이터를 체계적으로 저장하며 무결성, 보안, 동시성을 보장한다.

### 데이터베이스의 특징: 무결성, 일관성, 보안

- **무결성(Integrity)**: 데이터의 정확성과 신뢰성을 유지한다.
- **일관성(Consistency)**: 트랜잭션 완료 후 데이터베이스 상태가 일관된 상태로 유지하도록 보장한다.
- **보안(Security)**: 데이터 접근 권한을 제어하여 데이터 손실 및 유출 방지한다.
- **동시성(Concurrency)**: 여러 사용자가 동시에 데이터에 접근 가능하게 한다.
- **복구성(Recovery)**: 시스템 장애나 오류 발생 시 데이터를 복구할 수 있도록 지원한다.

### 데이터베이스 관리 시스템(DBMS)의 역할과 구조

- **DBMS(DataBase Management System)** 역할
    - 데이터 저장, 검색, 업데이트, 삭제와 같은 작업을 효율적으로 처리한다.
    - 데이터 무결성, 보안, 동시성, 복구를 지원한다.
    - SQL과 같은 언어를 통해 사용자와 데이터베이스 간 인터페이스를 제공한다.
- 구조
    - **사용자 인터페이스**: 데이터베이스와 상호작용하는 SQL 명령어 처리한다.
    - **질의 처리기**: 사용자 요청을 최적화하여 실행 계획 생성한다.
    - **트랜잭션 관리자**: 데이터의 일관성과 무결성을 보장한다.
    - **스토리지 관리자**: 데이터를 물리적으로 저장하고 관리한다.
    - **버퍼 관리자**: 메모리와 디스크 간 데이터를 효율적으로 관리한다.

### 데이터베이스 모델의 종류

- **관계형 데이터베이스**: 테이블 형식으로 데이터를 구조화 (예: MySQL, PostgreSQL).
- **비관계형 데이터베이스(NoSQL)**: 유연한 데이터 모델 제공 (예: MongoDB, Cassandra).
- **계층형/네트워크형 모델**: 트리나 그래프 구조로 데이터 표현.
